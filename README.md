# Simon Delaigue - CV

Online CV hosted on GitHub Pages.

## CI/CD

- `cv-check.yml`: validates the PDF and static files on `main`, `dev`, `feat/*`, `release/*`, `fix/*` + PRs targeting `main`, `dev`, `release/*`.
- `deploy.yml`: publishes to GitHub Pages on each push to `main`.
- `release.yml`: creates a GitHub Release on `cv-*` tags and attaches the PDF.

## Versions

- `VERSION`: CV document version (`cv-x.y.z`).
- `version.json`: metadata displayed on the page.
- `cvVersion`: CV version (from `VERSION`).
- `siteVersion`: technical site version (`site-<run_number>`, generated by CI).
- `deployedAt`: UTC deployment date.

This separation avoids confusing recruiters.

## Release

1. Update the version in the document and export the PDF.
2. Update `VERSION` with the same value (`cv-x.y.z`).
3. Merge to `main` (automatic check + deploy).
4. Create tag `cv-x.y.z` from `main` (must match `VERSION`).
